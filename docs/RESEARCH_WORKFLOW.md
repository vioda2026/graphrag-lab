# GraphRAG Research Workflow

## Goal
把“论文调研 → 创新提炼 → 代码实验”打通成闭环，避免只停留在二手总结。

## Required Loop (per new paper)

1. **发现与筛选**
   - 使用搜索工具收集候选（论文、博客、repo、他人总结）
   - 记录来源与时间，初筛相关性（GraphRAG / 图结构 / 图探索 / GraphRAGBench）

2. **原文定位与阅读（必做）**
   - 找到论文原文（PDF/官方页面）
   - 阅读并提取：问题定义、方法、核心假设、实验设置、主要结果、局限性

3. **逐篇笔记（Notion 子页面）**
   - 在 `GraphRAG项目` 下为该论文新建子页面
   - 页面模板建议：
     - 基本信息（标题/作者/年份/链接）
     - 一句话贡献
     - 方法拆解（图结构、图探索、训练/推理）
     - 实验与benchmark（是否含GraphRAGBench）
     - 可复现点与风险
     - 对我方任务的启发

4. **分类与关联**
   - 分类标签：任务类型、图结构策略、图探索策略、评测基准
   - 与现有论文建立关系：
     - 相近方法
     - 对立/替代方法
     - 可组合方法

5. **创新提炼 → 实验任务化**
   - 从本篇提炼 1-3 个可执行创新点
   - 写入 `实验记录（Ablation）`：
     - 结构-only
     - 探索-only
     - 联合策略
   - 标注优先级、预期收益、验证指标

## Cadence
- 每 6 小时：研究巡检与增量更新
- 每天 22:00：工作汇总，包含论文进展与实验联动状态

## Principle
- 先事实（原文证据）后判断（创新建议）
- 研究记录必须可追溯（来源、链接、时间）
- 结论必须可实验（能落到代码与评测）
